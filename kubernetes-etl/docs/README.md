# Архитектура решения (Knowledge Graph)

## Обзор
Система предназначена для поиска и анализа связей между статьями, экспериментами, материалами, свойствами, режимами, установками, командами и выводами.

## Выбранный стек технологий
- **Хранилище:** PostgreSQL. Реляционная СУБД отлично подходит для хранения структуры графа (Nodes и Edges) и выполнения аналитических SQL-запросов (включая CTE).
- **Трансформации:** DBT (Data Build Tool). Инструмент собирает сырые данные и формирует удобную для поиска витрину-граф.
- **Оркестрация/Среда выполнения:** Kubernetes (k3s / Gcloud). Обеспечивает работу БД и запуск аналитических джобов DBT.

## Модель данных (Граф)
В базе формируются две ключевые таблицы:
1. **`nodes` (Узлы):** Все сущности системы (статьи, эксперименты, сплавы X, режимы Y, свойства Z и т.д.).
2. **`edges` (Связи):** Связи между узлами (например, Эксперимент `ИСПОЛЬЗУЕТ` Материал, Эксперимент `ПРОВОДИТСЯ_В_РЕЖИМЕ` Режим).

Над этими таблицами создано представление `kg_search_view`, которое объединяет данные для ответов на бизнес-вопросы, такие как: _«что уже делали по сплавам X при режиме Y и какой был эффект на свойство Z»_.

С помощью DBT эта структура формируется из корпуса внутренних документов и каталога экспериментов автоматически.
